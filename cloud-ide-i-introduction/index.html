Cloud IDE I - Introduction

I am always obsessed with the idea of a portable development environment. (Note: Not necessarily IDE.) The reason is that development environment is very personal. There are so many possible permutations - IDE vs non-IDE, auto-completion vs non-auto-completion, OS X vs Linux vs Windows, the choice of language, the choice of text editor and so on and so forth. I started fulfilling my “dream” by making Ubuntu Live USB with persistent memory, configure Gedit and Terminal to suit me. But as time goes and when I was loaned a Chromebook, I thought, “Hey, come on, it’s the age of Cloud. Seriously? Live USB?”. So I went on and look for Cloud IDE. It wasn’t easy. Not much I found. These are the stuff that I expect to have:

  • Shell access
  • Decent text editor
  • Git access (direct integration with GitHub and BitBucket is optional)
  • (optional) Root access for shell
  • SSH from shell to other machine
  • SSH from other machine to shell
  • (optional) Web server
  • Supports these languages:
    • C++
    • (optional) PHP
    • Ruby (on Rails)
    • Python
    • (optional) Go

With these requirements, only three left: Nitrous.IO, Koding and Cloud9. I shall review each of these in details in the coming articles. Stay tuned.
